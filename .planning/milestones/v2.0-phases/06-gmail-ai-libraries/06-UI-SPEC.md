## Copywriting Contract

Source: Existing components and error-handling from Phase 3/4. Phase 6 adds no new user-visible copy. The contract below documents the existing copy that backend output shapes must support.

| Element | Copy |
|---------|------|
| Primary CTA — email summary | "Copy All Summaries" |
| Primary CTA — LinkedIn post (Workflow 2 & 3) | "Copy Post" |
| Reset action — email summary | "New Summary" |
| Reset action — post workflows | "New Post" |
| Loading — copy pattern | Workflow-specific stage messages (existing, unchanged) |
| Empty state | Not applicable — Phase 6 produces no empty UI. If backend returns empty array for Workflow1Data, the existing `ResultDisplay` renders an empty card with no items — no new empty state needed. |
| Error — auth expired | "Your session has expired. Please sign in again." (existing, from Phase 4 error handling) |
| Error — timeout | "This is taking longer than expected." (existing) |
| Error — empty result | "No results found. Try a different date range or topic." (existing) |
| Error — generic | "Something went wrong. Please try again." (existing) |
| Destructive actions | None in Phase 6. |

**Backend output copy requirements:**

- `Workflow2Data` (string): Plain prose LinkedIn post text only. No trailing newlines beyond paragraph breaks. No markdown. No "Here is your post:" preamble.
- `Workflow3Data` personal path (string): Same — plain prose, no preamble, no markdown formatting.
- `Workflow3ResearchData.summary`: Plain prose paragraph. No markdown headings.
- `EmailSummaryItem.summary`: 2-4 sentence prose summary per sender. No markdown.
- `EmailSummaryItem.keyTopics`: Array of short label strings (1-4 words each). No punctuation.
- `EmailSummaryItem.actionableInsights`: Array of imperative sentences ("Consider X", "Review Y"). No markdown bullets in the strings — the UI renders its own `<li>` elements.
- `generateQuestions()` output: Numbered plain-text list. Example: "1. What was the main challenge?\n2. What did you learn?". No markdown.
