## Threat Model

### T-01: Gmail access token leakage (HIGH)
- **Threat:** accessToken logged or exposed in error messages
- **Mitigation:** Never log the accessToken value. Error messages must use generic text ("Gmail API error: {status}"), not include the token.
- **Verification:** `grep -r "accessToken" src/lib/gmail.ts` shows only parameter declarations and Authorization header usage, never console.log with the token value.

### T-02: Tavily API key exposure (MEDIUM)
- **Mitigation:** TAVILY_API_KEY read from process.env at module scope. Never logged. SDK handles key injection internally.
- **Verification:** `grep "TAVILY_API_KEY" src/lib/search.ts` shows only the process.env read, not any logging.

### T-03: Server-Side Request Forgery via Gmail API (LOW)
- **Mitigation:** Gmail API URL is hardcoded (`https://gmail.googleapis.com/gmail/v1/users/me/messages`). No user-supplied URL components.
- **Verification:** URL construction uses `new URL()` with hardcoded base, user input only in `q` query parameter.

### src/lib/gmail.ts

Create `src/lib/gmail.ts` with the following implementation per D-07, D-08, D-09, D-10:

**Types to define at top of file:**

```typescript
export interface EmailMessage {
  id: string
  sender: string       // email address extracted from "From" header
  senderName: string   // display name from "From" header (e.g., "Hamel Husain")
  subject: string
  date: string         // from "Date" header
  body: string         // plain text, HTML stripped
}

export type GroupedEmails = Map<string, EmailMessage[]>
```

**Internal Gmail API response types (not exported):**

```typescript
interface GmailPart {
  mimeType: string
  body?: { data?: string }
  parts?: GmailPart[]
}

interface GmailMessage {
  id: string
  payload: {
    headers: Array<{ name: string; value: string }>
    mimeType: string
    body?: { data?: string }
    parts?: GmailPart[]
  }
}

interface GmailListResponse {
  messages?: Array<{ id: string }>
  nextPageToken?: string
}
```

**Exported function signature:**

```typescript
export async function fetchEmails(
  accessToken: string,
  startDate: Date,
  endDate: Date
): Promise<GroupedEmails>
```

**Implementation details:**

1. **`formatDateForGmail(date: Date): string`** — Internal helper. Converts Date to `YYYY/MM/DD` format (slashes, NOT dashes — per Pitfall 1 in RESEARCH.md). Use `date.getFullYear()`, `date.getMonth() + 1`, `date.getDate()` with zero-padding.

2. **`listMessageIds(accessToken: string, query: string): Promise<string[]>`** — Internal function. Calls `GET https://gmail.googleapis.com/gmail/v1/users/me/messages` with:
   - `q` parameter: the Gmail query string
   - `maxResults`: `500`
   - `pageToken`: from previous response if paginating
   - Header: `Authorization: Bearer ${accessToken}`
   - Sequential pagination per D-08: loop while `nextPageToken` exists
   - On non-OK response: throw `new Error(\`Gmail list failed: ${res.status}\`)`
   - Do NOT log the accessToken in error messages (T-01)

3. **`getMessage(accessToken: string, id: string): Promise<GmailMessage>`** — Internal function. Calls `GET https://gmail.googleapis.com/gmail/v1/users/me/messages/${id}?format=full` with Bearer auth header. Returns the parsed JSON.

4. **`extractBody(part: GmailPart): string | null`** — Internal recursive function per Pattern 2 in RESEARCH.md:
   - If `part.mimeType === "text/html"` and `part.body?.data` exists: decode with `Buffer.from(part.body.data, "base64url").toString("utf-8")` (base64url, NOT base64 — per Pitfall 2)
   - If `part.parts` exists: recurse into children, return first non-null result
   - If `part.mimeType === "text/plain"` and `part.body?.data` exists: decode as fallback
   - Return `null` if nothing found

5. **`htmlToText(html: string): string`** — Internal function per D-07, Pattern 3 in RESEARCH.md. Zero dependencies, regex-only:
   - Remove `<script>` and `<style>` blocks: `.replace(/<(script|style)[^>]*>[\s\S]*?<\/\1>/gi, " ")`
   - Strip remaining tags: `.replace(/<[^>]+>/g, " ")`
   - Decode HTML entities: `&nbsp;` -> ` `, `&amp;` -> `&`, `&lt;` -> `<`, `&gt;` -> `>`, `&quot;` -> `"`, `&#39;` -> `'`, catch-all `&[a-z]+;` -> ` `
   - Collapse whitespace: `.replace(/\s+/g, " ").trim()`

6. **`parseEmailHeader(headers: Array<{name: string; value: string}>, name: string): string`** — Internal helper. Finds header by case-insensitive name match, returns value or empty string.

7. **`parseSender(fromHeader: string): { email: string; name: string }`** — Internal helper. Parses `"Display Name <email@example.com>"` format. If no angle brackets, treat entire string as email. Extract email with regex like `/\<([^>]+)\>/` and name as the part before `<`.

8. **`fetchEmails` implementation:**
   - Build query: `category:updates after:${formatDateForGmail(startDate)} before:${formatDateForGmail(endDate)}`
   - Call `listMessageIds(accessToken, query)`
   - For each message ID, call `getMessage(accessToken, id)` — sequentially (per D-08, single-user app)
   - For each message: extract body via `extractBody(message.payload)`, convert HTML to text via `htmlToText()`, truncate to 8000 characters max per email body (per Pitfall 6 in RESEARCH.md — token budget guard)
   - Parse sender from "From" header, subject from "Subject" header, date from "Date" header
   - Build `EmailMessage` objects
   - Group into `Map<string, EmailMessage[]>` keyed by sender email address (per D-10)
   - Return the Map
