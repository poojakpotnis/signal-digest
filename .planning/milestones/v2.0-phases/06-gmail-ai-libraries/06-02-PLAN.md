## Threat Model

### T-01: Anthropic API key leakage (HIGH)
- **Threat:** ANTHROPIC_API_KEY logged or exposed in error messages
- **Mitigation:** SDK reads key from env automatically. Never log it. Error messages use generic text.
- **Verification:** `grep -n "ANTHROPIC_API_KEY" src/lib/claude.ts` shows zero matches (SDK reads from env internally, no explicit env access needed).

### T-02: Prompt injection via email content (MEDIUM)
- **Threat:** Malicious newsletter content could attempt to manipulate AI output by embedding directives in email body text
- **Mitigation:** Email content is placed in the user message, not the system prompt. System prompt is hardcoded. Claude's system prompt takes precedence over user message injected instructions by design.
- **Verification:** All system prompts are string literals (not interpolated from user input). User content only in `messages[].content`.

### T-03: JSON parse failure leading to untyped data (MEDIUM)
- **Mitigation:** Retry mechanism (D-05). If retry also fails, throw an error rather than returning malformed data. Caller (Phase 7 route handler) catches and returns error response.
- **Verification:** `callWithJsonRetry` throws on second parse failure (no silent fallback to unvalidated data).
</threat_model>

### Internal helper: callWithJsonRetry

Per D-05: prompt-based JSON output with retry. No Zod.

```typescript
async function callWithJsonRetry<T>(
  systemPrompt: string,
  userMessage: string,
  model: string,
  maxTokens: number = 4096
): Promise<T>
```

Implementation per Pattern 5 in RESEARCH.md:
1. Call `anthropic.messages.create({ model, max_tokens: maxTokens, system: systemPrompt, messages: [{ role: "user", content: userMessage }] })`
2. Extract text: `response.content.find(b => b.type === "text")?.text ?? ""`
3. Strip markdown code fences: `.replace(/^```(?:json)?\n?/, "").replace(/\n?```$/, "").trim()`
4. Try `JSON.parse(jsonStr) as T` — if success, return
5. On parse failure, retry ONCE: call `anthropic.messages.create` with conversation history including the malformed response and a correction message: `"Your response was not valid JSON. Please respond with ONLY valid JSON matching the schema, no other text."`
6. Strip fences from retry response, parse, return
7. If retry also fails, `throw new Error("Failed to parse JSON response from Claude after retry")`
