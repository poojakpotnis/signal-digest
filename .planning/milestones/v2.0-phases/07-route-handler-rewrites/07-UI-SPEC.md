## Copywriting Contract

Phase 7 rewrites route handlers only. The following copy contracts are inherited from Phase 3/4 and must not regress. Error messages returned by the new route handlers must match these exactly so `useWorkflow` surfaces them correctly.

| Element | Copy |
|---------|------|
| Primary CTA | "Generate" (all three workflow pages — unchanged) |
| Loading state | Contextual cycling messages per workflow (unchanged — rendered by `LoadingState` component) |
| Empty state heading | "No results found" |
| Empty state body | "No emails were found in the selected date range. Try expanding your date range and generating again." |
| Auth expired error | "Your session has expired. Please sign in again to continue." |
| Timeout error | "This is taking longer than expected. Please try again." |
| Generic error | "Something went wrong. Please try again." |
| Empty result (Workflow 1) | "No emails found in this date range." |
| Invalid input (400) | Route handler returns `{ error: "..." }` with specific field name — frontend `useWorkflow` surfaces as generic error state |
| Destructive confirmation | None — Phase 7 contains no destructive user-facing actions |

Error response shape contract (route handler output, consumed by `useWorkflow.handleResponse()`):

- Success: `{ data: <WorkflowNData> }` — use `data` field wrapper for clean unwrapping
- Q&A stage: `{ stage: "questions", data: <string[]>, resumeUrl: "/api/followup" }` — exact shape required by `useWorkflow` line 38 check
- Error: `{ error: <string> }` with appropriate HTTP status

Source: CONTEXT.md decisions D-04, D-05, D-06, D-07; REQUIREMENTS.md INT-02; `src/hooks/use-workflow.ts` (line 38 check noted in CONTEXT.md)

## Phase 7 Implementation Notes for Executor

These are behavioral constraints derived from INT-02 (zero UI changes):

1. Route handlers must return `{ data: result }` wrapper — `useWorkflow.handleResponse()` tries `json.data` first.
2. Personal post Q&A path must return `{ stage: "questions", data: questions, resumeUrl: "/api/followup" }` — `resumeUrl` field must be present (truthy gate at line 38 of use-workflow.ts).
3. Error codes in JSON responses must remain string-only (`auth_expired`, `timeout`, `empty_result`, `generic` are the frontend-consumed values — map server errors to these via HTTP status codes per D-06).
4. No new UI components, pages, or layout changes permitted in this phase.
5. Loading indicators, result display, and error states all render from existing `LoadingState` and `ResultDisplay` components — no modifications to these files.
