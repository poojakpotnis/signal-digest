# Requirements: LinkedIn Content Generator

**Defined:** 2026-03-30
**Core Value:** A single authorized user can sign in with Google and generate publish-ready LinkedIn posts in under 60 seconds, without writing from scratch.

## v2.0 Requirements

Requirements for self-contained AI backend. Each maps to roadmap phases.

### Auth

- [x] **AUTH-01**: User's Google OAuth refresh token is stored in the session so Gmail access survives beyond the 1-hour access token expiry

### Gmail

- [ ] **GMAIL-01**: App fetches emails from the Gmail Updates tab within a user-specified date range, with pagination handling for >100 results
- [ ] **GMAIL-02**: Email HTML bodies are converted to plain text before processing to reduce token usage and improve AI output quality

### AI Generation

- [ ] **AI-01**: User can generate a structured email summary grouped by sender using Claude API directly (replacing n8n Workflow 1)
- [ ] **AI-02**: User can generate a LinkedIn post from email insights with optional influencer research using Claude + Tavily directly (replacing n8n Workflow 2)
- [ ] **AI-03**: User can generate a LinkedIn post on any custom topic using Tavily search + Claude directly (replacing n8n Workflow 3 research path)
- [ ] **AI-04**: User can generate a personal experience LinkedIn post through an interactive Q&A flow powered by Claude directly (replacing n8n Workflow 3 personal path)

### Integration

- [ ] **INT-01**: Route handlers `/api/generate` and `/api/followup` call Gmail/Claude/Tavily directly instead of proxying to n8n webhooks
- [ ] **INT-02**: All existing frontend components, hooks, and response shapes remain unchanged — zero UI modifications required
- [ ] **INT-03**: New environment variables `ANTHROPIC_API_KEY` and `TAVILY_API_KEY` replace `N8N_WEBHOOK_URL` and `N8N_FOLLOWUP_WEBHOOK_URL`

## Future Requirements

Deferred to future release. Tracked but not in current roadmap.

### Performance

- **PERF-01**: Gmail messages fetched in parallel batches for faster workflow execution
- **PERF-02**: Claude prompt caching enabled for repeated prompt patterns
- **PERF-03**: Per-workflow model routing (Haiku for summarization, Sonnet for generation)

### Features

- **FEAT-01**: Sender filtering using curated Supabase newsletter list
- **FEAT-02**: Multi-user access control with role-based permissions

## Out of Scope

| Feature | Reason |
|---------|--------|
| Modifying frontend UI | Response shapes preserved — no UI changes needed |
| Building n8n replacement dashboard | Direct API calls are simpler than replicating n8n's UI |
| Direct LinkedIn publishing | Copy-to-clipboard only, as in v1.0 |
| Supabase sender list integration | Deferred — not needed for core workflow replacement |
